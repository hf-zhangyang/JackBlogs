# 博客文章管理指南

本文档说明如何在技术博客中添加新的文章分类、子分类和文章。

## 📂 项目结构

```
docs/
├── articles/
│   ├── articles.js          # 文章数据源（核心配置文件）
│   ├── backend/            # 后端开发分类文章
│   ├── docker/             # Docker 分类文章
│   ├── linux/              # Linux 运维分类文章
│   └── index.md            # 博客列表页面
└── .vitepress/
    └── theme/
        └── components/
            └── ArticlesList.vue  # 博客列表渲染组件
```

## 🎯 数据源文件

**核心文件**：`docs/articles/articles.js`

所有文章数据都定义在这个文件的 `articleList` 数组中，博客列表页面会自动读取并渲染。

## 📝 添加新内容的三个级别

### 级别 1：添加新分类（一级目录）

在 `articleList` 数组中添加新的分类对象：

```javascript
{
  category: "分类名称",      // 必填：显示在页面上的分类标题
  icon: "📘",               // 必填：分类图标（emoji）
  description: "分类描述",   // 必填：分类的简介文字
  subcategories: [          // 必填：子分类数组
    {
      name: "子分类名称",
      items: [
        // 文章列表
      ]
    }
  ]
}
```

**示例**：添加"数据库"分类

```javascript
export const articleList = [
  // ... 现有分类

  {
    category: "数据库",
    icon: "🗄️",
    description: "MySQL、Redis、MongoDB 等数据库的使用和优化。",
    subcategories: [
      {
        name: "MySQL",
        items: [
          {
            title: "MySQL 性能优化",
            link: "/articles/database/mysql-optimize",
            desc: "索引优化、查询优化、配置调优"
          }
        ]
      }
    ]
  }
]
```

### 级别 2：在现有分类中添加子分类（二级目录）

在现有分类的 `subcategories` 数组中添加新的子分类对象：

```javascript
{
  category: "容器与运维",
  icon: "📘",
  description: "Docker 容器化部署、镜像优化、网络配置与多服务编排。",
  subcategories: [
    {
      name: "Docker",
      items: [...]
    },
    // 新增子分类
    {
      name: "Kubernetes",
      items: [
        {
          title: "K8s 入门教程",
          link: "/articles/k8s/intro",
          desc: "Kubernetes 基础概念和部署"
        }
      ]
    }
  ]
}
```

### 级别 3：添加新文章（三级内容）

**步骤 1**：创建 Markdown 文件

在对应目录下创建 `.md` 文件，例如：
```
docs/articles/docker/nginx.md
```

**步骤 2**：在 `articles.js` 中添加文章条目

在对应子分类的 `items` 数组中添加：

```javascript
{
  name: "Docker",
  items: [
    {
      title: "Docker 部署最佳实践",
      link: "/articles/docker/deploy",
      desc: "基础镜像优化、数据持久化与编排"
    },
    // 新增文章
    {
      title: "Nginx 反向代理配置",
      link: "/articles/docker/nginx",
      desc: "Nginx 容器化部署与反向代理配置"
    }
  ]
}
```

## 📋 文章对象字段说明

每个文章对象包含三个字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | ✅ | 文章标题，显示在列表中 |
| `link` | string | ✅ | 文章链接路径，必须对应实际文件位置 |
| `desc` | string | ✅ | 文章描述，显示在标题下方 |

## 🔗 链接路径规则

**重要**：`link` 字段必须与实际文件路径对应

```
link: "/articles/docker/nginx"
  ↓ 对应文件
docs/articles/docker/nginx.md
```

**路径规则**：
- 以 `/articles/` 开头
- 然后是分类目录名
- 然后是文件名（不含 `.md` 扩展名）
- 使用 `/` 分隔路径层级

## ⚠️ 常见错误

### 错误 1：链接 404

**问题**：点击文章链接显示 404

**原因**：`link` 路径与实际文件路径不匹配

**解决**：检查 `link` 是否正确，确保文件存在

```javascript
// ❌ 错误
link: "/articles/docker/nginx"
// 实际文件在：docs/articles/container/nginx.md

// ✅ 正确
link: "/articles/container/nginx"
```

### 错误 2：内容不显示

**问题**：添加后页面不显示新内容

**原因**：可能是语法错误或 `articles.js` 未保存

**解决**：
1. 检查 JavaScript 语法是否正确
2. 确保逗号、括号配对
3. 刷新浏览器时按 Ctrl + Shift + R 强制刷新

### 错误 3：分类显示乱序

**问题**：分类顺序不符合预期

**解决**：调整 `articleList` 数组中的元素顺序

## 📚 批量操作建议

如果要添加大量文章：

1. **先规划好分类结构**
2. **一次性创建所有目录和文件**
3. **最后统一更新 `articles.js`**
4. **测试验证所有链接**

## 🔧 维护建议

1. **定期备份** `articles.js` 文件
2. **保持命名规范**：文件名使用小写字母和连字符
3. **及时更新**：添加文章后立即更新 `articles.js`
4. **保持一致性**：描述文字风格保持统一

## 📞 技术支持

如遇到问题，检查：
1. 浏览器控制台是否有错误
2. `articles.js` 语法是否正确
3. 文件路径是否匹配
4. 是否强制刷新了浏览器
